Spring 2025 Semester

getstaRted

Learning Objectives:

  1. Understand the basics of RStudio, including the purpose of the four panes.
  2. Learn how to execute R code and interpret the output in the Console.
  3. Explore built-in functions in R.
  4. Introduce the concept of objects, vectors, and data frames in R.
###############  Let's get staRted! ###############

# This file is called an R script (a file containing R code)
# Notice that this script is called "getstaRted.R" (see tab above)
# Think of R files (.R) as text files that contain R code

# Hashtags are used before text comments to explain what the code is for 
## You can put more than one if you want!

# There are 4 panes in the R Studio window
# This pane (top left) is the Editor pane, where you can edit the code

# To submit code, put your cursor *anywhere on the line* and either:
# - hit ctrl+enter for PC or cmd+enter for Mac users
# - click "Run" on the top right of this pane
print("Welcome to R!")
## [1] "Welcome to R!"
# Notice how your output popped out on the bottom pane: this is the Console!

#------------------------------------------------------------------------------#
# R is a calculator
#------------------------------------------------------------------------------#

6*2 # 6 times 2
## [1] 12
6/2 # 6 divided by 2
## [1] 3
6^2 # 6 squared
## [1] 36
##### Try it! #####
# Multiply 6 by all previous whole numbers (except 0). 
# Note: This is called the factorial of 6.
6*5*4*3*2*1
## [1] 720
#------------------------------------------------------------------------------#
# R has built-in functions + documentation
#------------------------------------------------------------------------------#

# Base R comes with many built-in functions:
factorial(6) # better to use this function for calculating factorials!
## [1] 720
##### Try it! #####
# Find the factorial of 100 using the built-in function.


# What should we expect for log(100)? What does R give us?
log(100)
## [1] 4.60517
# If you would like to know more about a function: 
# Put a `?` in front of the function name
?log

# Note: the documentation appears in the Output pane!
##### Try it! #####
# In the documentation you can read about the function that computes log base 10.
# What is log base 10 of 100?
log10(100)
## [1] 2
#------------------------------------------------------------------------------#
# R saves and uses objects
#------------------------------------------------------------------------------#

# The pane in the top right is the Environment. 
# This is where objects (like datasets) will go!
# There's nothing there yet because we haven't created any objects

# By convention, the notation "<-" is an assignment operator
# Note: You can also use "=" but it is not common to do so in the R community

# Let's consider some values and save them as objects
variety <- "Fuji"
sweetness <- 1
red <- FALSE

# IMPORTANT
# Object names... 
# start with a letter, 
# shouldn’t contain spaces, 
# should not be predefined in R (e.g., do not label an object "log")

# Now do you see these objects in the Environment? 
# It saves what is on the right as an object whose name is on the left
variety
## [1] "Fuji"
# Objects in R can be of different types.
# Use class() to check what type of values we saved in the environment
class(variety)
## [1] "character"
class(sweetness)
## [1] "numeric"
class(red)
## [1] "logical"
# We can save several values in one object called a vector
# Combine elements with c() which stands for concatenate
varieties <- c("Fuji", 'Gala', 'Golden Delicious', 'Granny Smith') 
varieties
## [1] "Fuji"             "Gala"             "Golden Delicious"
## [4] "Granny Smith"
# Note that either ' ' or " " can be used to define strings (with characters)

# Vector with numeric values
sweetness <- 1:4 # what does ":" do?
sweetness
## [1] 1 2 3 4
# Let's play around with the sweetness object
# Add up all the values in the sweetness using the sum() function
sum(sweetness)
## [1] 10
# How many values does our "sweetness" vector contain?
length(sweetness)
## [1] 4
##### Try it! #####
# What is the mean value of the sweetness?
sum(sweetness)/length(sweetness)
## [1] 2.5
# but easier with mean(), a built-in function!
mean(sweetness)
## [1] 2.5
# What if we add some more data in sweetness?
c(sweetness, 5)
## [1] 1 2 3 4 5
# What if this data is of a different type?
c(sweetness, "very sweet")
## [1] "1"          "2"          "3"          "4"         
## [5] "very sweet"
# Check the type of data now
class(c(sweetness, "very sweet"))
## [1] "character"
# R coerced the numeric values into characters
# What if we want to coerce sweetness as a numeric vector? 
as.numeric(c(sweetness, "very sweet")) # what happened?
## Warning: NAs introduced by coercion
## [1]  1  2  3  4 NA
# We can save several vectors in one object called a data frame
# A data frame is a very important data structure for most relational data
sweet_apples <- data.frame(
  varieties = c('Fuji', 'Gala', 'Golden Delicious', 'Granny Smith'),
  sweet_index = c(1, 2, 2, 4),
  red = c(TRUE,TRUE,FALSE,FALSE)
)
sweet_apples

# Rows represent different observations, columns represent different variables
str(sweet_apples) # Use str() to get more information about an object
## 'data.frame':    4 obs. of  3 variables:
##  $ varieties  : chr  "Fuji" "Gala" "Golden Delicious" "Granny Smith"
##  $ sweet_index: num  1 2 2 4
##  $ red        : logi  TRUE TRUE FALSE FALSE
# You can access the different variables with $
sweet_apples$varieties
## [1] "Fuji"             "Gala"             "Golden Delicious"
## [4] "Granny Smith"
##### Try it! #####
# Calculate the mean sweetness in this data frame
mean(sweet_apples$sweet_index)
## [1] 2.25
